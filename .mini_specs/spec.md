# Especificação de Tarefas

## Instruções

Sua missão aqui é traduzir planos em tarefas detalhadas na área de devops.

Analise os topicos abaixo e, se houver alguma dúvida, faça perguntas de esclarecimento.

Após lacunas sanadas, construa `.mini_specs/tasks.md` conforme o seguinte:
- as tarefas (em formato de checklist) para fase do plano
- um sumário dinamico no inicio, apenas 1 nível e usando link 'a' com id como âncora para cada fase.
- Via de regra, as tarefas serão feitas por voce mesmo. Mas Se houver alguma tarefa ou fase em que eu precise fazer algo, deixe claro na task.
- seja objetivo, conciso e claro
- use formatacao markdown adequada para deixar o documento legível e didático

Finalizado o documento, solicite minha revisão. Uma vez aprovada, iniciamos a execucao.

A medida que as tasks forem sendo concluídas, iremos marcando como feitas.

Obs: adicione algo nessa linha no inicio de tasks.md:
```
**CRÍTICO**: Siga o seguinte ciclo para cada fase:
> implemente uma fase → testa "manual" (o q fizer sentido) → commita → atualiza tasks.md
obs: Inclusive, se necessário, pode fazer testes em passos intermediários dentro da propria fase. 
```

Pode sobre escrever o arquivo tasks.md sem problema. Só garanta que ele está em UTF-8 e você vai escrever em UTF-8.
